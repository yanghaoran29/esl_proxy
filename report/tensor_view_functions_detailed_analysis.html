# Tensor View函数深度分析报告 ## 1. 概述 本报告详细分析三个核心Tensor操作函数：`tensor_make_2d`、`tensor_view` 和 `tensor_view_2d`。这三个函数构成了PTO运行时张量视图系统的核心，提供了从创建到切片查看的完整能力。 ### 1.1 函数定位 所有函数均位于：`esl_proxy/include/tensor.h` - **tensor_make_2d**: 第143-149行 - **tensor_view**: 第152-160行 - **tensor_view_2d**: 第163-174行 --- ## 2. Tensor数据结构 在分析函数之前，必须理解底层Tensor结构： ```cpp struct Tensor { // === Cache line 1 (64B) — hot path === uint64_t buffer_addr; // 缓冲区基地址 uint64_t buffer_size; // 缓冲区总大小 uint64_t owner_task_id; // 所有者任务ID uint64_t start_offset; // 数据起始偏移（字节） int32_t version; // 版本号 uint32_t ndims; // 维度数 uint8_t dtype; // 数据类型 uint8_t manual_dep; // 手动依赖标记 uint8_t is_contiguous; // 是否连续 uint8_t _pad_cl1; uint32_t shapes[5]; // 各维度大小 // === Cache line 2 (64B) — warm path === uint64_t extent_elem_cache; // 元素范围缓存 uint32_t strides[5]; // 各维度步长 uint8_t _pad_cl2[36]; } __attribute__((aligned(64))); ``` **关键设计特点**： - 总大小：128字节（2个缓存行） - 对齐：64字节对齐，优化缓存访问 - 双缓存行设计：热路径和温路径分离 --- ## 3. 子函数详解 ### 3.1 tensor_fill_row_major_strides **功能**：计算行主序步长数组 ```cpp static inline void tensor_fill_row_major_strides(uint32_t ndims, const uint32_t shapes[], uint32_t strides_out[]) { uint32_t s = 1; for (int32_t i = (int32_t)ndims - 1; i >= 0; i--) { strides_out[i] = s; s *= shapes[i]; } } ``` **算法逻辑**： 1. 初始化步长 `s = 1` 2. 从最后一维向前遍历 3. 每维步长等于后续所有维度大小的乘积 4. 更新累积值 `s *= shapes[i]` **示例**： ``` shapes = [3, 4, 5] (3D张量) strides = [20, 5, 1] 计算过程： i=2: strides[2] = 1, s = 1*5 = 5 i=1: strides[1] = 5, s = 5*4 = 20 i=0: strides[0] = 20, s = 20*3 = 60 ``` **时间复杂度**：O(n)，空间复杂度：O(1) --- ### 3.2 tensor_extent_elem_hull **功能**：计算张量的元素范围外壳 ```cpp static inline uint64_t tensor_extent_elem_hull(const Tensor *t) { uint64_t e = 1; for (int32_t i = (int32_t)t->ndims - 1; i >= 0; i--) { if (t->shapes[i] > 0u) { e += (uint64_t)(t->shapes[i] - 1u) * (uint64_t)t->strides[i]; } } return e; } ``` **数学含义**： 给定形状和步长，计算包含所有元素的最小范围 **公式**： ``` extent = 1 + Σ(shape[i] - 1) × stride[i] ``` **示例**： ``` shapes = [3, 4], strides = [4, 1] extent = 1 + (3-1)×4 + (4-1)×1 = 1 + 2×4 + 3×1 = 1 + 8 + 3 = 12 ``` **用途**： - 计算张量所需内存边界 - 验证视图操作的有效性 - 缓存计算结果避免重复计算 --- ### 3.3 tensor_refresh_derived **功能**：刷新派生字段（连续性标记和元素范围） ```cpp static inline void tensor_refresh_derived(Tensor *t) { uint64_t expected = 1; uint8_t contig = 1; for (int32_t i = (int32_t)t->ndims - 1; i >= 0; i--) { if (t->strides[i] != expected) { contig = 0; } expected *= t->shapes[i]; } t->is_contiguous = contig; t->extent_elem_cache = tensor_extent_elem_hull(t); } ``` **连续性判断逻辑**： 1. 从最后一维向前遍历 2. 检查每维步长是否符合行主序连续存储 3. 行主序连续条件：`stride[i] == shape[i+1] × shape[i+2] × ... × shape[ndims-1]` **示例**： ``` 连续张量 shapes=[3,4,5], strides=[20,5,1] i=2: stride[2]=1, expected=1 ✓ i=1: stride[1]=5, expected=1×5=5 ✓ i=0: stride[0]=20, expected=5×4=20 ✓ is_contiguous = 1 非连续张量 shapes=[3,4,5], strides=[20,4,1] i=2: stride[2]=1, expected=1 ✓ i=1: stride[1]=4, expected=1×5=5 ✗ is_contiguous = 0 ``` **调用时机**： - `tensor_make_contiguous` 创建新张量后 - `tensor_view` 和 `tensor_view_2d` 切片操作后 --- ### 3.4 tensor_make_contiguous **功能**：创建连续的n维张量 ```cpp static inline Tensor tensor_make_contiguous(uint64_t base, uint64_t buffer_size, const uint32_t shapes[], uint32_t ndims, dtype_t dtype) { Tensor t; memset(&t, 0, sizeof t); t.buffer_addr = base; t.buffer_size = buffer_size; t.start_offset = 0; t.version = 0; t.owner_task_id = 0; t.ndims = ndims; t.dtype = (uint8_t)dtype; t.manual_dep = 0; t.is_contiguous = 1; for (uint32_t i = 0; i < ndims; i++) { t.shapes[i] = shapes[i]; } tensor_fill_row_major_strides(ndims, t.shapes, t.strides); tensor_refresh_derived(&t); return t; } ``` **参数说明**： - `base`: 底层缓冲区基地址 - `buffer_size`: 缓冲区总大小（字节） - `shapes[]`: 各维度大小数组 - `ndims`: 维度数 - `dtype`: 数据类型（BFLOAT16=2, FLOAT32=4） **执行流程**： 1. 零初始化Tensor结构 2. 设置基础字段（地址、大小、类型等） 3. 复制形状数组 4. 计算行主序步长 5. 刷新派生字段 --- ## 4. 主函数详解 ### 4.1 tensor_make_2d **功能**：创建二维连续张量 ```cpp static inline Tensor tensor_make_2d(uint64_t base, uint32_t d0, uint32_t d1, dtype_t dtype) { const uint32_t shapes[2] = {d0, d1}; const uint64_t buffer_size = (uint64_t)d0 * (uint64_t)d1 * (uint64_t)dtype; return tensor_make_contiguous(base, buffer_size, shapes, 2, dtype); } ``` **参数**： - `base`: 缓冲区基地址 - `d0`: 第一维大小（行数） - `d1`: 第二维大小（列数） - `dtype`: 数据类型 **返回值**：二维连续Tensor对象 **使用示例**： ```cpp // 创建 100x200 的 BFLOAT16 张量 uint64_t buffer = allocate_buffer(100 * 200 * 2); // 2 = sizeof(bfloat16) Tensor t = tensor_make_2d(buffer, 100, 200, BFLOAT16); // 结果： // shapes = [100, 200] // strides = [200, 1] // 行主序 // is_contiguous = 1 // buffer_size = 40000 bytes ``` **应用场景**： - 初始化新的权重矩阵 - 创建输入/输出张量 - 分配临时计算缓冲区 **性能特性**： - 时间复杂度：O(ndims) = O(2) - 零拷贝：仅创建元数据，不复制数据 - 内联函数：编译器优化消除函数调用开销 --- ### 4.2 tensor_view **功能**：沿指定维度切片创建视图 ```cpp static inline Tensor tensor_view(Tensor t, uint32_t dim, uint32_t off, uint32_t n) { if (dim < t.ndims) { t.start_offset += (uint64_t)off * (uint64_t)t.strides[dim]; t.shapes[dim] = n; tensor_refresh_derived(&t); } return t; } ``` **参数**： - `t`: 输入张量 - `dim`: 要切片的维度索引 - `off`: 起始偏移（元素个数） - `n`: 切片后的维度大小 **操作语义**： 1. 计算新起始位置：`start_offset += off × stride[dim]` 2. 更新形状：`shapes[dim] = n` 3. 刷新派生字段（连续性、范围） **使用示例**： ```cpp // 原始张量: shapes=[10, 20], strides=[20, 1] Tensor t = ...; // 提取第一维的第3-7个元素（5个元素） // off=3, n=5 Tensor view = tensor_view(t, 0, 3, 5); // 结果：shapes=[5, 20], start_offset += 3*20 = 60 // 提取第二维的第10-14个元素（5个元素） // off=10, n=5 Tensor view2 = tensor_view(t, 1, 10, 5); // 结果：shapes=[10, 5], start_offset += 10*1 = 10 ``` **组合使用**： ```cpp // 先按第一维切片，再按第二维切片 Tensor t = ...; // shapes=[10, 20] Tensor v1 = tensor_view(t, 0, 2, 3); // shapes=[3, 20] Tensor v2 = tensor_view(v1, 1, 5, 10); // shapes=[3, 10] // 最终视图：t[2:5, 5:15] ``` **边界检查**： - `dim >= ndims` 时不执行任何操作（原样返回） - 不检查 `off + n` 是否超出原始边界 **性能特性**： - 时间复杂度：O(1) - 零拷贝：仅修改元数据 - 元数据复制：输入张量未被修改（按值传递） --- ### 4.3 tensor_view_2d **功能**：一次完成二维切片（单步操作） ```cpp static inline Tensor tensor_view_2d(Tensor t, uint32_t off0, uint32_t off1, uint32_t n0, uint32_t n1) { if (t.ndims >= 2) { t.start_offset += (uint64_t)off0 * (uint64_t)t.strides[0] + (uint64_t)off1 * (uint64_t)t.strides[1]; t.shapes[0] = n0; t.shapes[1] = n1; tensor_refresh_derived(&t); } return t; } ``` **参数**： - `t`: 输入张量（至少2维） - `off0`: 第一维偏移 - `off1`: 第二维偏移 - `n0`: 第一维切片大小 - `n1`: 第二维切片大小 **操作语义**： ```cpp t.start_offset += off0 × stride[0] + off1 × stride[1] t.shapes[0] = n0 t.shapes[1] = n1 ``` **等价操作**： ```cpp // tensor_view_2d 等价于： Tensor temp = tensor_view(t, 0, off0, n0); Tensor result = tensor_view(temp, 1, off1, n1); return result; ``` **使用示例**： ```cpp // 原始张量: shapes=[100, 200], strides=[200, 1] Tensor t = ...; // 提取子矩阵 t[10:20, 30:40]（10x10） Tensor sub = tensor_view_2d(t, 10, 30, 10, 10); // 计算： // start_offset = 10*200 + 30*1 = 2030 // shapes = [10, 10] ``` **实际应用案例**（来自qwen3_dynamic_tensormap.h）： ```cpp // 注意力机制的KV缓存切片 Tensor q_piece = tensor_view_2d(q_proj, // 原始Q投影结果 (uint32_t)b0, // batch维度偏移 base * 256u, // sequence维度偏移 16u, // head维度大小 256u); // sequence切片大小 Tensor k_piece = tensor_view_2d(k_proj, (uint32_t)b0, base * 128u, 16u, 128u); ``` **性能优势**： - 相比连续调用两次 `tensor_view`： - 减少一次函数调用 - 减少一次 `tensor_refresh_derived` 调用 - 只需一次派生字段计算 **边界检查**： - `t.ndims < 2` 时不执行任何操作 --- ## 5. 函数关系图 ``` ┌─────────────────────────────────────────────────────────────────┐ │ tensor_make_2d (L143) │ │ 功能: 创建新的2D张量 │ │ 调用链: │ │ └── tensor_make_contiguous (L120) │ │ ├── tensor_fill_row_major_strides (L101) │ │ └── tensor_refresh_derived (L87) │ │ └── tensor_extent_elem_hull (L76) │ └─────────────────────────────────────────────────────────────────┘ ┌─────────────────────────────────────────────────────────────────┐ │ tensor_view (L152) │ │ 功能: 单维度切片视图 │ │ 调用链: │ │ └── tensor_refresh_derived (L87) │ │ └── tensor_extent_elem_hull (L76) │ └─────────────────────────────────────────────────────────────────┘ ┌─────────────────────────────────────────────────────────────────┐ │ tensor_view_2d (L163) │ │ 功能: 二维同时切片视图 │ │ 调用链: │ │ └── tensor_refresh_derived (L87) │ │ └── tensor_extent_elem_hull (L76) │ └─────────────────────────────────────────────────────────────────┘ ``` --- ## 6. 数据流示例 ### 6.1 创建张量并切片 **场景**：创建权重矩阵并提取批次 ```cpp // Step 1: 创建 1024x4096 的权重矩阵 uint64_t weight_buffer = allocate_buffer(1024 * 4096 * 2); // BFLOAT16 Tensor weights = tensor_make_2d(weight_buffer, 1024, 4096, BFLOAT16); // shapes=[1024, 4096], strides=[4096, 1] // Step 2: 提取batch=5的权重向量 // 等价于 weights[5, :]（整个第二维） Tensor batch_weights = tensor_view(weights, 0, 5, 1); // shapes=[1, 4096], start_offset=5*4096=20480 // Step 3: 进一步切片，只取前512列 Tensor partial = tensor_view(batch_weights, 1, 0, 512); // shapes=[1, 512], start_offset=20480 // 验证派生字段 printf("extent_elem_cache = %lu\n", partial.extent_elem_cache); // extent = 1 + (1-1)×4096 + (512-1)×1 = 512 ``` ### 6.2 Attention机制中的使用 ```cpp // Q, K, V 投影结果: [batch, seq_len, heads, head_dim] // 实际存储为连续2D: [batch×seq_len×heads, head_dim] Tensor q_proj = tensor_make_2d(q_buffer, total_q, head_dim, BFLOAT16); // shapes=[total_q, head_dim] // 为每个batch和head提取Q块 for (int b = 0; b < batch; b++) { for (int h = 0; h < heads; h++) { int offset = (b * heads + h) * q_len; Tensor q_block = tensor_view_2d( q_proj, offset, // 第一维偏移 0, // 第二维偏移 q_len, // 第一维大小 head_dim // 第二维大小 ); // q_block 指向 q_proj[offset:offset+q_len, 0:head_dim] // 执行注意力计算... } } ``` --- ## 7. 内存布局详解 ### 7.1 Tensor结构内存分布 ``` 字节偏移 字段 大小 缓存行 ────────────────────────────────────────────── 0 buffer_addr 8 CL1 (热) 8 buffer_size 8 CL1 16 owner_task_id 8 CL1 24 start_offset 8 CL1 32 version 4 CL1 36 ndims 4 CL1 40 dtype 1 CL1 41 manual_dep 1 CL1 42 is_contiguous 1 CL1 43 _pad_cl1 1 CL1 44 shapes[0] 4 CL1 48 shapes[1] 4 CL1 52 shapes[2] 4 CL1 56 shapes[3] 4 CL1 60 shapes[4] 4 CL1 ────────────────────────────────────────────── 64 extent_elem_cache 8 CL2 (温) 72 strides[0] 4 CL2 76 strides[1] 4 CL2 80 strides[2] 4 CL2 84 strides[3] 4 CL2 88 strides[4] 4 CL2 92 _pad_cl2[0-35] 36 CL2 ────────────────────────────────────────────── 总计: 128字节 (2 × 64字节缓存行) ``` ### 7.2 步长与寻址 **一维索引公式**： ``` linear_index = Σ(shape[i] × stride[i]) ``` **二维索引公式**： ``` element_addr = base + start_offset + i × stride[0] × dtype_size + j × stride[1] × dtype_size ``` **示例**： ```cpp Tensor t = tensor_make_2d(base, 100, 200, BFLOAT16); // shapes=[100, 200], strides=[200, 1] // 访问 t[35, 87] uint64_t linear_idx = 35 * 200 + 87 * 1; // 7097 uint64_t byte_offset = 7097 * 2; // BFLOAT16=2字节 void *elem_ptr = (void *)(base + byte_offset); ``` ### 7.3 视图的内存语义 ```cpp // 原张量 Tensor t = tensor_make_2d(base, 100, 200, BFLOAT16); // start_offset = 0 // 视图1: off0=10, off1=20 Tensor v1 = tensor_view_2d(t, 10, 20, 30, 40); // start_offset = 10*200*2 + 20*1*2 = 4040 bytes // 逻辑上等同于 t[10:40, 20:60] // 视图2: 在v1基础上再切片 Tensor v2 = tensor_view(v1, 0, 5, 10); // start_offset = 4040 + 5*200*2 = 6040 bytes // 逻辑上等同于 t[15:25, 20:60] ``` **关键点**： - 视图只修改 `start_offset` 和 `shapes` - 不复制底层数据 - 多层视图的偏移是累积的 --- ## 8. 性能优化建议 ### 8.1 选择正确的视图函数 | 场景 | 推荐函数 | 原因 | |------|---------|------| | 创建新张量 | `tensor_make_2d` | 一次调用完成所有初始化 | | 单维度切片 | `tensor_view` | 通用性强 | | 多维度同时切片 | `tensor_view_2d` | 减少函数调用开销 | | 维度≥3的切片 | 多次 `tensor_view` | 无专用函数 | ### 8.2 避免不必要的派生刷新 ```cpp // 低效：每次切片都刷新派生字段 Tensor v1 = tensor_view_2d(t, off0, off1, n0, n1); Tensor v2 = tensor_view(v1, 2, off2, n2); // 再次刷新 // 优化：如果不需要派生字段，可以延迟刷新 // （当前实现总是刷新，无法避免） ``` ### 8.3 批量视图创建 ```cpp // 低效：为每个元素创建独立视图 for (int i = 0; i < n; i++) { Tensor view = tensor_view(base_tensor, dim, offsets[i], sizes[i]); process(view); // 处理后view可能不再使用 } // 优化：复用视图对象 Tensor view = base_tensor; // 初始化 for (int i = 0; i < n; i++) { view = tensor_view(base_tensor, dim, offsets[i], sizes[i]); process(view); } ``` --- ## 9. 常见错误与调试 ### 9.1 越界访问 ```cpp // 错误：偏移+大小超出原始范围 Tensor t = tensor_make_2d(base, 10, 20, BFLOAT16); // shapes=[10, 20] // 危险：off1=15, n1=10，超出原始20的范围 Tensor v = tensor_view_2d(t, 0, 15, 5, 10); // 可能访问到 t[:, 15:25]，但原始只有20列 ``` **症状**： - 读取垃圾数据 - 写入破坏其他数据 - 编译器不报错，难以发现 **调试方法**： ```cpp static inline Tensor tensor_view_2d_safe(Tensor t, uint32_t off0, uint32_t off1, uint32_t n0, uint32_t n1) { if (t.ndims < 2) { fprintf(stderr, "Error: tensor has less than 2 dimensions\n"); exit(1); } if (off0 + n0 > t.shapes[0]) { fprintf(stderr, "Error: dim0 slice out of bounds (%u + %u > %u)\n", off0, n0, t.shapes[0]); exit(1); } if (off1 + n1 > t.shapes[1]) { fprintf(stderr, "Error: dim1 slice out of bounds (%u + %u > %u)\n", off1, n1, t.shapes[1]); exit(1); } return tensor_view_2d(t, off0, off1, n0, n1); } ``` ### 9.2 维度混淆 ```cpp // 错误：混淆维度顺序 Tensor t = tensor_make_2d(base, rows, cols, dtype); // shapes=[rows, cols] // 错误地认为 strides[0] = cols, strides[1] = 1 // 实际上：strides[0] = cols, strides[1] = 1（行主序） // 当访问 t[i, j] 时 uint64_t offset = i * t.strides[0] + j * t.strides[1]; // i * cols + j * 1 ``` ### 9.3 类型转换错误 ```cpp // 错误：dtype使用不当 Tensor t = tensor_make_2d(base, 100, 200, 2); // 应该用 BFLOAT16 // 编译器可能不报错，但语义不清晰 // 正确：使用枚举常量 Tensor t = tensor_make_2d(base, 100, 200, BFLOAT16); // 检查dtype定义 typedef enum { BFLOAT16 = 2, FLOAT32 = 4, } dtype_t; ``` --- ## 10. 总结 ### 10.1 函数对比 | 特性 | tensor_make_2d | tensor_view | tensor_view_2d | |------|---------------|-------------|----------------| | 功能 | 创建新张量 | 单维度切片 | 二维同时切片 | | 调用子函数 | 2个 | 1个 | 1个 | | 时间复杂度 | O(ndims) | O(1) | O(1) | | 内存操作 | 元数据初始化 | 仅修改偏移 | 仅修改偏移 | | 适用场景 | 初始化 | 灵活切片 | 高效2D切片 | ### 10.2 设计哲学 1. **零拷贝视图**：所有视图函数仅修改元数据，不复制数据 2. **缓存行优化**：Tensor结构按缓存行对齐（64字节） 3. **内联优化**：所有函数声明为 `static inline`，消除调用开销 4. **派生字段缓存**：计算并缓存连续性和范围信息，避免重复计算 5. **简洁接口**：最小化参数数量，提供专用函数减少用户错误 ### 10.3 使用建议 - **初始化**：使用 `tensor_make_2d` 创建新的张量对象 - **切片**： - 单维度 → `tensor_view` - 二维度 → `tensor_view_2d` - 多维度 → 多次 `tensor_view` - **组合**：可以链式调用多个视图操作 - **调试**：在关键路径添加边界检查 --- **报告生成时间**：`tensor_view_2d` 函数分析完成 **代码版本**：esl_proxy/include/tensor.h **分析深度**：子函数追踪 + 内存布局 + 性能优化
